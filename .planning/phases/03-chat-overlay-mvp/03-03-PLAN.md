3. `handleKeyDown` callback:
   - If `e.key === 'Enter'` and NOT `e.shiftKey`:
     - `e.preventDefault()`
     - If `value.trim()` is not empty:
       - Call `onSend(value + '\r')` — MUST be `\r` not `\n` (per Pitfall 1: ConPTY expects carriage return for command execution)
       - `setValue('')`
       - `textareaRef.current?.focus()` (per D-03: preserves input box focus after send)
   - Shift+Enter: do nothing special — default textarea behavior inserts newline.

4. Auto-expand textarea: Use an `onChange` handler that adjusts textarea height:
   ```typescript
   const handleChange = useCallback((e: React.ChangeEvent<HTMLTextAreaElement>) => {
     setValue(e.target.value);
     // Auto-expand: reset height to auto, then set to scrollHeight, capped at 4 lines (~6rem)
     const textarea = e.target;
     textarea.style.height = 'auto';
     textarea.style.height = Math.min(textarea.scrollHeight, 96) + 'px'; // 96px ~ 4 lines at 14px font
   }, []);
   ```

5. JSX structure:
   ```tsx

       onKeyDown={handleKeyDown}
   ```
