<objective>
Add SQLite session persistence to the Node.js sidecar using better-sqlite3. Every PTY session records its output as chunked BLOBs with batched writes (500ms / 64KB flush). Session metadata is persisted for later replay.

Purpose: This is the data layer for HIST-01 (in-session capture) and HIST-02 (cross-session persistence). The history sidebar (Plan 04) will query this data. Without the batching layer, high-frequency PTY output would stutter the terminal.

Output: historyStore.ts (schema + queries), sessionRecorder.ts (batched writer), ptySession.ts and server.ts extended with recording integration.
</objective>

   ```typescript

   ```

   DB path: `path.join(process.env['LOCALAPPDATA'] || path.join(os.homedir(), 'AppData', 'Local'), 'chat-overlay-widget', 'sessions.db')`.

   Export `openDb()`: creates directory with `mkdirSync(path.dirname(dbPath), { recursive: true })`, opens DB, sets `pragma('journal_mode = WAL')` and `pragma('synchronous = NORMAL')`, creates tables:
   ```sql
   CREATE TABLE IF NOT EXISTS sessions (
     id         INTEGER PRIMARY KEY AUTOINCREMENT,
     shell      TEXT    NOT NULL,
     cwd        TEXT,
     started_at INTEGER NOT NULL,
     ended_at   INTEGER,
     is_orphan  INTEGER DEFAULT 0
   );
   CREATE TABLE IF NOT EXISTS session_chunks (
     id          INTEGER PRIMARY KEY AUTOINCREMENT,
     session_id  INTEGER NOT NULL REFERENCES sessions(id),
     data        BLOB    NOT NULL,
     recorded_at INTEGER NOT NULL
   );
   CREATE INDEX IF NOT EXISTS idx_chunks_session ON session_chunks(session_id, recorded_at);
   ```
   Store the db instance in a module-level variable.

   Export `getDb()`: returns the module-level db instance. Throws if `openDb()` was not called.

   Export `markOrphans()`: runs `UPDATE sessions SET is_orphan = 1 WHERE ended_at IS NULL` (per D-17). Call this at startup before creating new sessions.

   Export `listSessions()`: returns `db.prepare('SELECT id, shell, cwd, started_at, ended_at, is_orphan FROM sessions ORDER BY started_at DESC').all()` typed as `SessionRow[]`.

   Export `getSessionChunks(sessionId: number)`: returns `db.prepare('SELECT data FROM session_chunks WHERE session_id = ? ORDER BY recorded_at ASC').all(sessionId)` typed as `{ data: Buffer }[]`.

   Export interface `SessionRow`:
   ```typescript
   export interface SessionRow {
     id: number;
     shell: string;
     cwd: string | null;
     started_at: number;
     ended_at: number | null;
     is_orphan: number;
   }
   ```

3. Create `sidecar/src/sessionRecorder.ts`:

   ```typescript
   import { getDb } from './historyStore.js';
   ```

   Export class `SessionRecorder`:
   - Private fields: `sessionId: number`, `buffer: Buffer[]`, `bufferSize: number`, `flushTimer: NodeJS.Timeout | null`, `insertChunkStmt` (prepared statement), `ended: boolean`.
   - Constants: `FLUSH_INTERVAL = 500` (ms), `FLUSH_SIZE = 65536` (64KB).
   - Constructor takes `shell: string, cwd: string`:
     - Gets db via `getDb()`.
     - Inserts session row: `INSERT INTO sessions (shell, cwd, started_at) VALUES (?, ?, ?)` with `Date.now()`.
     - Stores `lastInsertRowid` as `sessionId`.
     - Prepares the chunk insert statement once: `INSERT INTO session_chunks (session_id, data, recorded_at) VALUES (?, ?, ?)`.
     - Starts interval: `this.flushTimer = setInterval(() => this.flush(), this.FLUSH_INTERVAL)`.
   - `append(data: string): void`: converts data to Buffer, pushes to buffer array, increments bufferSize. If bufferSize >= FLUSH_SIZE, calls `this.flush()`.
   - Private `flush(): void`: if buffer is empty, return. `Buffer.concat(this.buffer)`, reset buffer and bufferSize, run prepared statement with `(this.sessionId, combined, Date.now())`.
   - `end(): void`: calls `flush()`, clears interval, updates session: `UPDATE sessions SET ended_at = ? WHERE id = ?` with `(Date.now(), this.sessionId)`. Sets `ended = true`.
   - `destroy(): void`: if not ended, best-effort flush. Clears interval. Does NOT set ended_at (orphan per D-17).
   - Getter `getSessionId(): number` returns sessionId.
