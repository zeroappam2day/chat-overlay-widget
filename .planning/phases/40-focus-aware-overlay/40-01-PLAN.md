Create `sidecar/src/win32Bridge.ts` per D-01, D-03, D-04:

**Class: Win32Bridge**
- `init()`: Spawn `powershell.exe -NoProfile -NoExit -Command -` via `child_process.spawn`. Send the Add-Type C# block once on stdin. The C# class (`Win32Focus`) must declare:
  - `[DllImport("user32.dll")] static extern IntPtr GetForegroundWindow();`
  - `[DllImport("user32.dll")] static extern uint GetWindowThreadProcessId(IntPtr hWnd, out uint pid);`
  - `[DllImport("user32.dll")] static extern bool IsWindow(IntPtr hWnd);`
  - `[DllImport("user32.dll")] static extern bool IsIconic(IntPtr hWnd);`
  - `[DllImport("user32.dll", SetLastError=true)] static extern IntPtr GetWindow(IntPtr hWnd, uint uCmd);` (GW_OWNER = 4)

- **Stdin/stdout protocol**: Each request is a JSON line on stdin: `{"id":<number>,"cmd":"<name>","args":[...]}`. The PowerShell reads lines in a `while($true)` loop, dispatches by `cmd`, and writes a JSON line on stdout: `{"id":<number>,"result":<value>}` or `{"id":<number>,"error":"<msg>"}`. Use `---READY---` sentinel on stdout after Add-Type compiles to signal init complete. Delimiter: newline (`\n`). Parse stdout with a line-based buffer (split on `\n`, accumulate partial lines).

- **Commands** (PowerShell switch block):
  - `getForegroundWindow` → `[Win32Focus]::GetForegroundWindow().ToInt64()`
  - `getWindowThreadProcessId` → args[0] is hwnd, returns `{"threadId":<uint>,"pid":<uint>}`
  - `isWindow` → `[Win32Focus]::IsWindow([IntPtr]::new($args[0]))` → bool
  - `isIconic` → `[Win32Focus]::IsIconic([IntPtr]::new($args[0]))` → bool
  - `getOwnerWindow` → `[Win32Focus]::GetWindow([IntPtr]::new($args[0]), 4).ToInt64()` → number

- **Health / resilience** (per D-04):
  - Each request has a 3000ms timeout (`setTimeout`). On timeout: reject the promise, increment a failure counter.
  - On process `exit` or `error` event: reject all pending requests, set `alive = false`.
  - Auto-restart: If `alive = false` and `destroyed = false`, call `init()` again. Next request waits for re-init.
  - Warmup: `init()` is called at module load time (not lazily). Export a singleton `win32Bridge` instance.

- **Exported API** (all return `Promise`):
  - `getForegroundWindow(): Promise<number>`
  - `getWindowThreadProcessId(hwnd: number): Promise<{threadId: number; pid: number}>`
  - `isWindow(hwnd: number): Promise<boolean>`
  - `isIconic(hwnd: number): Promise<boolean>`
  - `getOwnerWindow(hwnd: number): Promise<number>`
  - `init(): Promise<void>` (explicit init, also called internally)
  - `destroy(): void` (kills process, rejects pending)

Create `sidecar/src/focusTracker.ts` per D-02, D-05 through D-16:

**Interface:**
```typescript
interface FocusTrackerOptions {
  onShow: () => void;     // overlay should show
  onHide: () => void;     // overlay should hide
  onTargetLost: () => void; // target hwnd no longer valid
  pollIntervalMs?: number;  // default 250
  hideDebounceMs?: number;  // default 150
}
```

**Class: FocusTracker**
- Constructor takes `FocusTrackerOptions`.
- `start(targetHwnd: number): void` -- Begins polling. Resolves target PID once via `getWindowThreadProcessId(targetHwnd)`. Stores `targetPid`. Sets `isTracking = true`.
- `stop(): void` -- Clears timeout, sets `isTracking = false`, clears debounce timer.
- `destroy(): void` -- Calls `stop()`.
- `addSelfHwnd(hwnd: number): void` -- Adds hwnd to a `Set<number>` of self-windows. Per D-06 and D-07, call this for the main Tauri window hwnd and the annotation-overlay hwnd.

**Poll loop** (per D-02, using `setTimeout` chains, NOT `setInterval`):
1. Call `getForegroundWindow()` to get `fgHwnd`.
2. **Self-check**: If `fgHwnd` is in the self-hwnd set, treat as "show" (per D-06, D-07).
3. **Direct match**: If `fgHwnd === targetHwnd`, treat as "show".
4. **Stale check** (per D-13): Call `isWindow(targetHwnd)`. If false: call `onTargetLost()`, set `isTracking = false`, return (don't schedule next poll).
5. **Minimized check** (per D-09): Call `isIconic(targetHwnd)`. If true: immediate hide (skip debounce), schedule next poll, return.
6. **Owner chain** (per D-05): Walk `getOwnerWindow(fgHwnd)` up to 5 levels. If any returned hwnd === targetHwnd, treat as "show".
7. **PID fallback** (per D-05, D-08): Call `getWindowThreadProcessId(fgHwnd)`. If `fgPid === targetPid`, check process name is NOT `ApplicationFrameHost.exe` (per D-05). If affiliated, treat as "show". To check process name: use `getWindowThreadProcessId` result's pid and resolve process name via a one-time PowerShell query or cache the ApplicationFrameHost PID at start. Simpler: just add a `getProcessName(pid): Promise<string>` to win32Bridge that runs `[System.Diagnostics.Process]::GetProcessById($pid).ProcessName`. Cache `ApplicationFrameHost` PIDs in a Set.
8. **Not affiliated**: Treat as "hide".

**Show/Hide emission** (per D-10, D-11, D-12, D-14):
- Track `lastEmitted: 'show' | 'hide' | null` to deduplicate.
- Show: If `lastEmitted !== 'show'`, call `onShow()` immediately (zero debounce per D-12). Set `lastEmitted = 'show'`. Clear any pending hide debounce timer.
- Hide: If `lastEmitted !== 'hide'`, start a 150ms debounce timer (per D-11). If timer fires without a show interrupting, call `onHide()`. Set `lastEmitted = 'hide'`.
- Minimized hide: Skip debounce, call `onHide()` immediately.

**Note**: Add `getProcessName(pid: number): Promise<string>` to win32Bridge.ts (add the PowerShell command and export). This is a small addition to Task 1's module -- add it here since the need was discovered during focusTracker design.
