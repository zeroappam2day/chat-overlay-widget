Wire overlay auto-show/hide to walkthrough lifecycle events in the annotationBridgeStore.

```typescript
setWalkthroughStep: (step) => {
  if (!useFeatureFlagStore.getState().guidedWalkthrough) return;
  emit('update-walkthrough-step', step).catch((err) => {
    console.warn('[annotation-bridge] Failed to emit walkthrough step:', err);
  });
},
```

Key facts:
- showOverlay() calls overlay.show() which is idempotent (safe to call when already visible)
- hideOverlay() calls overlay.hide() which is idempotent
- Neither showOverlay nor hideOverlay check the annotationOverlay feature flag (only toggleOverlay does)
- showOverlay also calls overlay.setIgnoreCursorEvents(true) for ghost/click-through mode
- The emit and show/hide are both async fire-and-forget with .catch -- consistent pattern
