Add a closing `}` brace at the very end of the file to close the `else` block.

**Why this structure:** The `require()` call at the top of the file — before any `import` statements — ensures no native addon (node-pty, better-sqlite3, sharp) is loaded in MCP mode. TypeScript compiles `import` statements into hoisted `require()` calls at the top of the output JS, but a literal `if` guard before them in the emitted JS will only work if we use `require()` for the MCP module. Since TypeScript preserves top-level `if` statements before the compiled imports, this pattern is safe.

**IMPORTANT:** The guard must use `require('./mcp-server.js')` (CommonJS require), NOT an `import` statement. The `.js` extension is required because TypeScript with module=Node16 compiles `.ts` to `.js` and requires explicit extensions in imports.

After the require, add a statement that prevents fallthrough. The mcp-server module will call `process.exit()` on its own, but to be defensive:
```typescript
if (process.argv[2] === 'mcp') {
  require('./mcp-server.js');
  throw new Error('mcp-server should not return');
}
```
This way the rest of server.ts (all the imports, SQLite init, PTY setup) is simply never reached. No `else` block needed — the throw acts as a hard stop.

  const dir = path.join(process.env['APPDATA'] ?? os.homedir(), 'chat-overlay-widget');
  const filePath = path.join(dir, 'api.port');

**HTTP helper — makes GET requests to sidecar (per D-15, D-16):**
```typescript
function sidecarGet(endpoint: string, token: string, port: number): Promise<{ status: number; headers: http.IncomingHttpHeaders; body: Buffer }> {
  return new Promise((resolve, reject) => {
    const req = http.get(`http://127.0.0.1:${port}${endpoint}`, {
      headers: { 'Authorization': `Bearer ${token}` },
    }, (res) => {
      const chunks: Buffer[] = [];
      res.on('data', (chunk: Buffer) => chunks.push(chunk));
      res.on('end', () => resolve({ status: res.statusCode ?? 500, headers: res.headers, body: Buffer.concat(chunks) }));
    });
    req.on('error', reject);
    req.setTimeout(10000, () => { req.destroy(); reject(new Error('Request timeout')); });
  });
}
```

**Error wrapper — reads discovery and calls sidecar, returns error text if unavailable (per D-18):**
```typescript
async function callSidecar(endpoint: string): Promise<{ status: number; headers: http.IncomingHttpHeaders; body: Buffer }> {
  let discovery: { port: number; token: string };
  try {
    discovery = readDiscovery();
  } catch {
    throw new Error('Chat Overlay Widget not running. Start the app first.');
  }
  try {
    return await sidecarGet(endpoint, discovery.token, discovery.port);
  } catch (err) {
    throw new Error(`Chat Overlay Widget not reachable: ${err instanceof Error ? err.message : String(err)}`);
  }
}
```

**Tool 1 — read_terminal_output (per D-12):**
Register with `server.tool()`. Name: `read_terminal_output`. Description: `Read the current terminal buffer from Chat Overlay Widget. Returns the most recent lines of terminal output with ANSI codes stripped and secrets scrubbed. Use 'since' cursor for pagination — pass the cursor value from a previous call to get only new lines.`

Zod schema: `{ lines: z.number().int().min(1).max(500).default(50).describe('Number of lines to return (1-500, default 50)'), since: z.number().int().optional().describe('Cursor from previous call — returns only lines after this point') }`

Handler: Call `callSidecar(/terminal-state?lines=${lines}&since=${since}&scrub=true)`. Parse JSON body. Return `{ content: [{ type: 'text', text: JSON.stringify(parsed, null, 2) }] }`. On error, return `{ content: [{ type: 'text', text: error.message }], isError: true }`.

**Tool 2 — query_session_history (per D-13):**
Name: `query_session_history`. Description: `Query historical terminal output from a past session stored in Chat Overlay Widget's SQLite database. Returns the most recent lines from the specified session with secrets scrubbed.`

Zod schema: `{ sessionId: z.number().int().describe('Session ID to query'), lines: z.number().int().min(1).max(500).default(100).describe('Number of lines to return (1-500, default 100)') }`

Handler: Call `callSidecar(/session-history?sessionId=${sessionId}&lines=${lines}&scrub=true)`. Parse JSON body. Return as text content. On error, return isError content.

**Tool 3 — capture_screenshot (per D-14):**
Name: `capture_screenshot`. Description: `Capture a screenshot of the Chat Overlay Widget window. Returns a PNG image with sensitive areas blurred. The screenshot shows the current state of the terminal and any UI elements.`

Zod schema: `{}` (no parameters — always blur, always current window per D-16).

Handler: Call `callSidecar(/screenshot?blur=true)`. Check status — if not 200, parse JSON error body and return as text error. If 200, the body is raw PNG binary. Base64-encode it: `body.toString('base64')`. Return: `{ content: [{ type: 'image', data: base64, mimeType: 'image/png' }] }`.
